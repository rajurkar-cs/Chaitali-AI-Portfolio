BigID to S3 Data Modeling Project

An AI-Ready Privacy Data Engineering Artifact

Overview

This project involved designing a logical data model to migrate sensitive data classified in BigID (stored in MongoDB) into Amazon S3 for downstream AI/analytics use. The goal was to ensure data accuracy, privacy compliance, and readiness for responsible AI applications.

Objective

To demonstrate the ability to model, map, and migrate privacy-tagged data for scalable, AI-compatible storage in Amazon S3.

Process

  • Analyzed BigID’s metadata structure (MongoDB document format).
  • Identified critical PII categories and regulatory tags.
  • Mapped source metadata into a logical schema suitable for flat storage in S3.
  • Built an ER diagram and defined attribute types, keys, and lineage mappings.
  • Validated structure for privacy law coverage (CCPA, CPRA, GDPR).

Tools & Technologies

  • BigID
  • MongoDB (source structure)
  • Amazon S3
  • Erwin Data Modeler
  • Draw.io for diagrams

Value Proposition

The project demonstrates practical skills in data modeling, privacy governance, and cloud migration—crucial areas in AI readiness. It highlights the ability to translate unstructured sensitive data into structured, AI-consumable formats while meeting legal compliance.
